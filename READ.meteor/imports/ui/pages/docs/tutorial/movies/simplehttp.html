Acquire and Transform HTTP Data

Have you ever sat through a highly rated movie only to conclude it wasn't so great after all? In this chapter, we will get to the bottom of this matter. Specifically, we will create movie datasets through multiple HTTP queries, combine these individual arrays into a single array, and visualize this data by extracting key properties from this array.

Before you begin

You will need to 1) sign up, 2) request and 3) acquire an API key from The Movie DB (TMDB).

Create Datasets from URLs

Create Top Movies (1) dataset. In the Data Editor tab, switch the dataset type to URL. Use the following URL (modified with your TMDB api key) to access the top movies dataset: https://api.themoviedb.org/3/movie/top_rated?api_key=yourtmdbapikey&language=en-US&page=1. Create Top Movies (2) and Top Movies (3) datasets in the same manner with the page parameter set to 2 and 3 respectively in the URL.

Concatenate & Transform Datasets

Create Combined Ratings data set. In the Data Editor tab, switch the dataset type to Transform. Choose Top Movies (1), Top Movies (2), and Top Movies (3) as the parent datasets. Copy the function from this gist and paste into transform function portion of Combined Ratings.

The transform function concatenates the three movie arrays, selects a few fields, and renames some of the selected fields.

Create Rating vs Popularity visualization. In the Data tab, choose Combined Ratings as your dataset and switch the Chart Type to Scatter. Populate the basic options for this chart from this gist. Populate the advanced options for this chart from this gist. Resize your chart by switching to the layout view.

Your shared dashboard will be automatically updated in the public link for the app.

Key Concepts

  • You can create a READ dataset from any URL that returns a JSON response.
  • You can create a transformed data set in READ. The transformed dataset can get its input from multiple parent datasets. These inputs can be combined and transformed in any way you wish based on the logic within the transform function.
  • READ dashboards are reactive. Changes made to the datasets and visualizations are reflected instantaneously in shared dashboards.
