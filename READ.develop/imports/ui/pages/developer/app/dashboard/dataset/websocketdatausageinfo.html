URL

URL must be a valid websocket URL which sends JSON.stringified data: i.e., if the websocket sends the string s, then JSON.parse(s) javascript function invocation must return valid JSON.

In this example, the websocket URL sends a (variable) string s once every 0.05 secs. READ's websocket dataset internally calls JSON.parse(s) once it receives the string, which returns a number which is valid JSON.
